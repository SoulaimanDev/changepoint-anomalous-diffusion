# Changepoint Detection in Anomalous Diffusion Trajectories

This repository contains a deep learning workflow for estimating changepoint positions in synthetic anomalous diffusion trajectories. Each trajectory is generated as a one-dimensional time series composed of two consecutive diffusion regimes. The learning task is to predict the temporal location where the first regime ends and the second one begins.

The project compares several neural network architectures on the same balanced dataset and reports both global performance and transition-wise behavior.

## Problem Setting

Anomalous diffusion describes particle motion that does not follow standard Brownian diffusion. In this project, each trajectory has a fixed length of `L = 100` and contains exactly one changepoint.

A trajectory is built from two segments generated by two different diffusion models. The model receives the trajectory as input and predicts the changepoint position as a regression target.

The diffusion models considered in this study are:

| Symbol | Diffusion model |
|---|---|
| `ATTM` | Annealed Transient Time Motion |
| `CTRW` | Continuous-Time Random Walk |
| `FBM` | Fractional Brownian Motion |
| `LW` | Lévy Walk |
| `SBM` | Scaled Brownian Motion |

Since five diffusion models are used and the same model is not repeated before and after the changepoint, the dataset contains `5 × 4 = 20` ordered transitions.

Examples of ordered transitions include:

```text
ATTM → CTRW
CTRW → LW
FBM → SBM
SBM → ATTM
```

## Dataset

The dataset is synthetically generated and balanced across all ordered transitions. This makes the evaluation fair because each model is tested on the same number of samples for every transition type.

| Split | Samples per transition | Ordered transitions | Total samples |
|---|---:|---:|---:|
| Training | 10,000 | 20 | 200,000 |
| Validation | 1,000 | 20 | 20,000 |
| Test | 10,000 | 20 | 200,000 |

Main dataset parameters:

```text
Trajectory length: 100
Minimum segment length: 20
Number of diffusion models: 5
Number of ordered transitions: 20
Training samples: 200,000
Validation samples: 20,000
Test samples: 200,000
```

## Models

### LSTM

The LSTM model processes each trajectory as an ordered sequence. Its recurrent layers summarize temporal dependencies and produce a regression output for the changepoint position.

### ConvLSTM

The ConvLSTM model introduces local temporal windows before recurrent processing. This allows the architecture to use short-range variations while still modeling sequence information.

### Transformer

The Transformer model relies on positional information and multi-head self-attention. This allows the network to compare different locations of the same trajectory and infer where the temporal behavior changes.

For this notebook, the Transformer input is represented with four channels:

```text
x(t)
dx(t)
|dx(t)|
dx(t)^2
```

The first channel is the normalized trajectory. The other channels describe local increments and their magnitude. This gives the attention mechanism more information about local motion while keeping the architecture as a Transformer model.

### ConvTransformer

The ConvTransformer model first extracts local features using convolutional layers and then applies attention blocks to model longer temporal relations. It is designed to combine local pattern detection with global sequence comparison.

## Evaluation

The changepoint detection task is evaluated as a regression problem. Predictions are converted back to temporal points and compared with the true changepoint positions.

The two reported metrics are:

```text
MAE  = mean absolute error
RMSE = root mean squared error
```

Both values are expressed in trajectory time points.

## Final Model Comparison

| Model | MAE | RMSE |
|---|---:|---:|
| ConvTransformer | 7.20 | 11.63 |
| LSTM | 7.72 | 11.71 |
| Transformer | 7.87 | 12.37 |
| ConvLSTM | 8.71 | 12.58 |

The ConvTransformer reaches the lowest global error. The LSTM and Transformer models remain close in performance, while the ConvLSTM has the largest error in this final comparison.

The Transformer result corresponds to the four-channel input representation used in `04_transformer_changepoint_detection.ipynb`.

## Transformer Transition-wise Results

The Transformer does not perform equally on all ordered transitions. Some pairs of diffusion models produce a clearer changepoint, while others are more difficult because the two regimes are harder to separate.

Lowest MAE transitions:

| Transition | MAE | RMSE |
|---|---:|---:|
| `CTRW → LW` | 3.52 | 6.80 |
| `LW → CTRW` | 3.72 | 7.03 |
| `ATTM → LW` | 5.79 | 10.16 |
| `LW → ATTM` | 6.01 | 10.13 |

Highest MAE transitions:

| Transition | MAE | RMSE |
|---|---:|---:|
| `SBM → FBM` | 12.15 | 16.02 |
| `SBM → ATTM` | 11.83 | 15.80 |
| `FBM → SBM` | 11.74 | 15.67 |
| `ATTM → SBM` | 11.45 | 15.72 |

These results show that the difficulty of the task depends strongly on the ordered transition. Transitions involving `SBM` and `FBM` are among the most difficult for the Transformer, while transitions between `CTRW` and `LW` are easier to localize.

## Main Observations

- The dataset is balanced across the 20 ordered diffusion-model transitions.
- The task is solved as changepoint-position regression.
- ConvTransformer obtains the best global result in this comparison.
- The Transformer achieves competitive performance when the input includes local increment-based features.
- Errors vary significantly depending on the pair of diffusion models before and after the changepoint.
- Transition-wise evaluation is important because global metrics alone hide model-specific strengths and weaknesses.
