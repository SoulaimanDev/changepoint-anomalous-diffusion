# Changepoint Detection in Anomalous Diffusion

Reproducible code for the Master's thesis **Detection and localization of changepoints in anomalous-diffusion trajectories using deep learning** by Soulaiman Chair (Universitat Politècnica de València, 2026).

The project studies one-dimensional synthetic trajectories generated with five AnDi models: ATTM, CTRW, FBM, Lévy walk (LW), and SBM. The supervised task is deliberately limited to:

1. deciding whether one explicit model transition is present; and
2. localizing that transition when it exists.

It does **not** identify the diffusion model or estimate its anomalous exponent in each segment.

## Experimental protocols

| Protocol | Length | Task | Train | Validation | Test |
|---|---:|---|---:|---:|---:|
| A | 100 | Localization, changepoint guaranteed | 200,000 | 20,000 | 200,000 |
| B | 100 | Binary detection + localization | 200,000 | 20,000 | 200,000 |
| B | 200 | ConvTransformer extension | 200,000 | 20,000 | 200,000 |

For Protocol B, every split is balanced: half of the trajectories contain a changepoint and half are generated by a single model. Positive samples are balanced over the 20 ordered model transitions. Negative samples are balanced over the five models.

The changepoint is restricted to 20-80% of the trajectory:

- `L=100`: `cp` in `[20, 80]`, increment index `cp_dx=cp-1` in `[19, 79]`;
- `L=200`: `cp` in `[40, 160]`, increment index `cp_dx=cp-1` in `[39, 159]`.

The binary models receive raw increments `dx = diff(X)`, standardized per trajectory inside the notebooks with `epsilon=1e-6`. This is an offline normalization: statistics are computed from the complete sequence.

## Architectures

- **LSTM**: recurrent baseline with two temporal layers.
- **CNN-LSTM**: Conv1D feature extractor and dilated residual blocks followed by LSTM layers. The original notebook filenames retain `convlstm`, but the Protocol B implementation is not a ConvLSTM cell.
- **Transformer**: learned positional encoding and three encoder blocks.
- **ConvTransformer**: local convolutional stem and residual blocks followed by three Transformer blocks.

All Protocol B networks have separate outputs for binary detection and a discrete distribution over valid changepoint positions. Localization loss is masked for negative samples. Reported point estimates use the expectation of the predicted distribution, not its argmax.

## Main thesis results

### Protocol B, L=100

Localization errors labelled `all` include every truly positive trajectory, including false negatives.

| Model | Threshold | Accuracy | Precision | Recall | F1 | FPR | FNR | MAE all | RMSE all |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| LSTM | 0.35 | 0.7328 | 0.7341 | 0.7300 | 0.7320 | 0.2645 | 0.2700 | 12.7331 | 16.3504 |
| CNN-LSTM | 0.25 | 0.7891 | 0.7980 | 0.7743 | 0.7860 | 0.1960 | 0.2257 | 7.6645 | 11.9183 |
| Transformer | 0.30 | 0.7712 | 0.8093 | 0.7095 | 0.7561 | 0.1672 | 0.2905 | 8.7414 | 12.5307 |
| ConvTransformer | 0.30 | 0.8037 | 0.8184 | 0.7806 | 0.7990 | 0.1732 | 0.2194 | 7.5370 | 11.5706 |

Transformer produces the fewest false alarms, whereas ConvTransformer gives the best observed balance between detection and localization. These values describe one main training run; they are not multi-seed estimates.

## Installation

Python 3.10 or 3.11 is recommended. Create an isolated environment, then install:

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# Linux/macOS: source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

`requirements.txt` defines compatible dependency ranges. For archival reproduction of a completed run, also retain the generated dataset summary and an exact environment snapshot:

```bash
python -m pip freeze > requirements-lock.txt
```

## Generate datasets

Protocol A (`L=100`, changepoint guaranteed):

```bash
python scripts/build_synthetic_changepoint_dataset.py \
  --output-dir data_synthetic_changepoint_andi \
  --length 100 --seed 42
```

Protocol B (`L=100`, balanced with/without changepoint):

```bash
python scripts/build_synthetic_with_without_changepoint_dx.py \
  --output-dir data_synthetic_with_without_changepoint_dx \
  --length 100 --seed 42
```

Protocol B extension (`L=200`):

```bash
python scripts/build_synthetic_with_without_changepoint_dx.py \
  --output-dir data_synthetic_with_without_changepoint_dx \
  --length 200 --seed 42
```

Add `--dry-run` to either command for a small structural check before generating the full dataset. Full generation is computationally and storage intensive.

## Run notebooks

Run notebooks `01-06` for Protocol A and `07-11` for the general Protocol B comparison. Notebooks `12-13` provide the dedicated `L=200` generation and ConvTransformer analysis used for the revised thesis. Paths are resolved relative to the repository; no user-specific directory is required.

Notebook 11 supports both thesis lengths. It defaults to `L=100`. Set the environment variable before starting Jupyter for `L=200`:

```bash
# PowerShell
$env:TRAJECTORY_LENGTH = "200"
jupyter lab
```

```bash
# Linux/macOS
TRAJECTORY_LENGTH=200 jupyter lab
```

At `L=200`, notebook 11 automatically applies the documented batch size (`128`), negative-class weight (`1.0`), valid localization mask, and validation threshold grid (`0.05` to `0.50` in steps of `0.025`).

The dedicated notebooks default to the safe `FAST` mode. For the complete `L=200` workflow in PowerShell:

```powershell
$env:TFM_RUN_MODE = "GLOBAL"
jupyter lab
```

Run notebook 12 first to generate the three `L=200` HDF5 splits, followed by notebook 13 for training and evaluation. Existing datasets are never overwritten unless `TFM_OVERWRITE=1` is explicitly set. Notebook 13 uses the full 100-epoch configuration when `TFM_GLOBAL_SHORT_TEST` is unset or `0`; set it to `1` for a one-epoch integration check.

In Colab, notebook 13 mounts Google Drive automatically and defaults to `/content/drive/MyDrive/TFM_L200`. Both local and Colab paths can be overridden without editing cells:

```powershell
$env:TFM_PROJECT_ROOT = "D:\path\to\TFM_L200"
```

## Verification

Run the lightweight generator tests without TensorFlow or `andi-datasets`:

```bash
python -m unittest discover -s tests -v
```

The tests check class balance, ordered-transition counts, HDF5 shapes, negative sentinels, increment indexing, and the 20-80% changepoint interval.

## Interpretation limits

- Model identity and anomalous exponent usually change together, so the current dataset does not isolate their causal effects.
- Architectures differ in parameter count, loss weights, checkpoint scores, and threshold grids.
- Only one main run is available; small differences must be treated descriptively.
- No PELT, window-based detector, RANDI, or other anomalous-diffusion baseline has been executed on the same splits.
- Per-trajectory normalization uses both sides of the offline sequence; training-set and causal normalization remain useful ablations.
- Synthetic results do not establish performance on experimental trajectories.

These limitations are part of the thesis conclusions, not hidden implementation details.
